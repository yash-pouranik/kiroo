Developer workflow

From local debugging to CI gates in four steps

01

Capture traffic

Record interactions using HTTP commands, kiroo proxy, or browser cURL import.

kiroo proxy --target http://localhost:3000
02

Snapshot a baseline

Save a contract baseline before large refactors or backend releases.

kiroo snapshot save v1-stable
03

Analyze drift

Compare versions and rank impact from low to critical severity.

kiroo analyze v1-stable current --fail-on high
04

Share artifacts

Generate OpenAPI and Postman outputs from real interactions for teammates and documentation.

kiroo export --format openapi --out openapi.json

Command surface

Daily-use commands at a glance

kiroo get|post|put|delete <url>

Execute and record API interactions

kiroo replay <id> / kiroo edit <id>

Re-run or tweak historical calls quickly

kiroo snapshot save|compare

Version contract states and inspect drift

kiroo analyze <tag1> <tag2>

Semantic severity analysis with optional AI

kiroo check <url> --status 200 --has token

CI-friendly zero-code endpoint assertions

kiroo bench <url> -n 100 -c 10

Concurrent load tests from terminal

kiroo export --format openapi
