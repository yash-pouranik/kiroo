Installation

Kiroo requires Node.js 18 or later.

npm install -g kiroo

Verify the installation:

kiroo --version

kiroo init

Initialize Kiroo in the current project directory. This creates the .kiroo/ directory with default configuration.

kiroo init

During setup, you'll be prompted to optionally store:

  • baseUrl — default server URL for all requests
  • GROQ_API_KEY — for AI-powered analysis
  • LINGODOTDEV_API_KEY — for translation features

Configuration is saved to .kiroo/config.json and environment variables to .kiroo/env.json.

HTTP Methods

Execute an HTTP request, display the response, and save the full interaction to your local history.

kiroo get <url>
kiroo post <url> [options]
kiroo put <url> [options]
kiroo delete <url> [options]
kiroo patch <url> [options]

Options

FlagDescription
-H, --header <key:value>Add a custom request header
-d, --data <data>Request body. Accepts JSON or shorthand key=value pairs
-s, --save <key=path>Extract a value from the response and save it as a variable

Examples

# Login and capture the JWT token
kiroo post /api/auth/login -d "email=user@test.com password=secret" --save token=data.token

# Use the saved token in subsequent requests
kiroo get /api/users -H "Authorization: Bearer {{token}}"

# Send raw JSON body
kiroo post /api/projects -d '{"name":"kiroo","public":true}'

kiroo list

Display a paginated table of all recorded interactions.

kiroo list [options]

Options

FlagDescriptionDefault
-n, --limit <number>Number of records to show10
-o, --offset <number>Number of records to skip0
--date <YYYY-MM-DD>Filter by date—
--url <path>Filter by URL path—
--status <code>Filter by HTTP status code—

Example

kiroo list -n 20 --status 200

kiroo replay

Re-execute a recorded interaction and display a diff between the original and new response.

kiroo replay <interaction-id>

The interaction ID is shown in kiroo list output.

Example

kiroo replay 2026-03-10T14-30-05-123Z

kiroo edit

Open a recorded interaction in your default editor. Modify the request (URL, headers, body), save and close — Kiroo will immediately re-execute the updated request.

kiroo edit <interaction-id>

Uses $EDITOR, falls back to VS Code, then nano.

kiroo import

Import a request from a cURL command. Useful for copying requests from browser DevTools.

kiroo import "<curl command>"

Example

kiroo import "curl https://api.example.com/users -H 'Authorization: Bearer tok123'"

kiroo proxy

Start a transparent proxy between your frontend and backend. All API traffic flowing through the proxy is automatically recorded — no manual CLI commands needed.

kiroo proxy --target <backend-url> [options]

Options

FlagDescriptionDefault
-t, --target <url>Backend server URL (required)—
-p, --port <port>Port for the proxy to listen on8080

Example

# Point your frontend to localhost:8080 instead of 3000
kiroo proxy --target http://localhost:3000 --port 8080

kiroo snapshot

Save, list, and compare versioned snapshots of your current interaction history.

Subcommands

CommandDescription
snapshot save <tag>Save current interactions as a named snapshot
snapshot listList all saved snapshots
snapshot compare <tag1> <tag2>Structural diff between two snapshots

Options for compare

FlagDescription
--analyzeInclude semantic severity analysis
--aiAdd AI-generated impact summary

Examples

kiroo snapshot save v1-stable
kiroo snapshot list
kiroo snapshot compare v1-stable v2-refactor --analyze --ai

kiroo analyze

Run semantic blast-radius analysis between two snapshots. Produces an endpoint-by-endpoint severity report detecting field renames, removed endpoints, type changes, and status code shifts.

kiroo analyze <tag1> <tag2> [options]

Options

FlagDescription
--jsonOutput machine-readable JSON (for CI)
--fail-on <severity>Exit non-zero if max severity ≥ threshold. Values: low, medium, high, critical
--aiGenerate Groq-powered impact summary
--model <model>Override the default AI model
--max-tokens <n>Max completion tokens for AI summary

Prerequisites

Set your API key before using --ai:

kiroo env set GROQ_API_KEY gsk_your_key_here

Examples

# CI gate — fail the pipeline on high-severity changes
kiroo analyze v1 v2 --fail-on high --json

# Developer review — get AI explanation
kiroo analyze before-refactor after-refactor --ai

kiroo check

Execute a request and validate the response against rules. Exits with code 1 on failure — designed for CI pipelines and quick smoke tests.

kiroo check <url> [options]

Options

FlagDescription
-m, --method <method>HTTP method (default: GET)
-H, --header <key:value>Custom headers
-d, --data <data>Request body
--status <code>Assert HTTP status code
--has <fields>Assert response contains these JSON fields (comma-separated)
--match <key=val>Assert exact value for a JSON field

Example

kiroo check /api/login -m POST -d "user=yash pass=123" --status 200 --has token,userId

kiroo bench

Load-test an endpoint with configurable concurrency. Reports latency percentiles, throughput, and error rates.

kiroo bench <url> [options]

Options

FlagDescriptionDefault
-m, --method <method>HTTP methodGET
-n, --number <n>Total requests10
-c, --concurrent <n>Concurrent workers1
-v, --verboseShow individual request details—
-H, --header <key:value>Custom headers—
-d, --data <data>Request body—

Example

kiroo bench /api/projects -n 100 -c 10 -v

kiroo export

Export recorded interactions as a Postman Collection (v2.1) or OpenAPI 3.0.3 specification. Schemas, examples, and server URLs are automatically inferred from recorded traffic.

kiroo export [options]

Options

FlagDescriptionDefault
-f, --format <fmt>postman or openapipostman
-o, --out <file>Output filename—
--title <title>API title (OpenAPI)—
--api-version <ver>API version (OpenAPI)—
--server <url>Server URL (OpenAPI)—

Examples

kiroo export --format postman --out my_collection.json
kiroo export --format openapi --out swagger.json --title "My API" --api-version 1.0.0

kiroo graph

Print a dependency tree showing how data flows between endpoints via --save and {{variable}} references.

kiroo graph

kiroo stats

Display summary statistics: total requests, success/failure rates, average latency, and the top 5 slowest endpoints.

kiroo stats

kiroo scrub

Retroactively redact sensitive values (auth headers, tokens, passwords, API keys) from all stored interactions and snapshots.

kiroo scrub [options]

Options

FlagDescription
--dry-runPreview what would change without writing
New interactions are redacted automatically at save time. Use scrub to clean data recorded before redaction was enabled.

kiroo env

Manage environments and variables. Sensitive values are masked when listing.

Subcommands

CommandDescription
env listList all variables (secrets masked)
env use <name>Switch active environment
env set <key> <value>Set a variable
env rm <key>Remove a variable

Examples

kiroo env set baseUrl https://api.myapp.com
kiroo env set GROQ_API_KEY gsk_your_key_here
kiroo env use production
kiroo env list

kiroo clear

Delete all recorded interactions.

kiroo clear [options]

Options

FlagDescription
-f, --forceSkip confirmation prompt
